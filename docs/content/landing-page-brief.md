## Tone of Voice Guidelines

### Brand Personality
- **Profesjonell men vennlig** - Ikke korporat eller stiv
- **Teknisk men tilgjengelig** - Bruk kode-referanser uten å ekskludere
- **Motiverende uten overhype** - Realistisk om verdien
- **Community-fokusert** - "Vi" over "du"

### Skrivestil
- Korte, punchige setninger
- Aktive verb ("Del kode" ikke "Kode kan deles")
- Emoji sparsomt og kun for visuell interesse
- Kode-snippets/referanser der det passer
- Norsk hovedspråk, men engelske tech-termer OK

### Eksempler på tone

| Unngå | Foretrekk |
|-------|-----------|
| "Verdens beste utviklerplattform" | "Der utviklere deler og lærer" |
| "Revolusjonerende AI-teknologi" | "AI-genererte utfordringer som skjerper ferdighetene" |
| "Uendelige muligheter" | "Snippets, diskusjoner, og challenges på ett sted" |

## 1. Hero Section

### Layout
- Full-width, mørk bakgrunn med subtil gradient
- Venstre: Tekst og CTA-knapper
- Høyre: Terminal/editor mockup med animert kode

### Content

**Badge/Eyebrow** (valgfritt):
```
// Open Source Community
```

**H1 Headline:**
```
Kode. Del. Løs. Gjenta.
```

**Alternative H1:**
```
Der utviklere skjerper ferdighetene
```

**Subheadline (1-2 setninger):**
```
Del kode-snippets, delta i tekniske diskusjoner, 
og utfordre deg selv med AI-genererte algoritme-challenges.
```

**Primary CTA Button:**
```
Kom i gang gratis →
```

**Secondary CTA Button:**
```
Utforsk challenges
```

### Bilde-behov

**Hero Visual:**
- Terminal/editor mockup som viser et stilig kode-snippet
- Mørkt tema (matcher app)
- Animert cursor eller syntax highlighting
- Placeholder: Screenshot fra app eller mockup i Figma
- Format: SVG/PNG, transparent bakgrunn der mulig

## 2. Features Grid

### Layout
- 3-kolonne grid på desktop, stacked på mobil
- Hvert feature-kort har ikon, tittel, og beskrivelse
- Subtil hover-effekt

### Content

**Section Header:**
```
H2: Alt du trenger på ett sted
Subtext: Fokuser på koding, ikke verktøy-hopping.
```

**Feature 1: Snippets**
```
Ikon: Code2 (fra Lucide)
H3: Kode-Snippets
Beskrivelse: Del og oppdag kode med syntax highlighting 
for 20+ språk. Tagg, lik, og lagre favoritter.
```

**Feature 2: Diskusjoner**
```
Ikon: MessageSquare (fra Lucide)
H3: Diskusjoner
Beskrivelse: Still spørsmål, del innsikt, og lær av 
community. Støtter markdown og nested svar.
```

**Feature 3: Challenges**
```
Ikon: Trophy (fra Lucide)
H3: Code Challenges
Beskrivelse: Test ferdighetene dine med AI-genererte 
algoritme-oppgaver. Easy, Medium, eller Hard - du velger.
```

**Feature 4: Gamification** (optional 4th)
```
Ikon: Star (fra Lucide)
H3: Poeng & Badges
Beskrivelse: Tjen poeng for bidrag og lås opp badges. 
Klatre på leaderboardet.
```

## 3. How It Works

### Layout
- Horisontalt stepper-design (1 → 2 → 3)
- Nummererte steg med ikon og beskrivelse
- Mørk bakgrunn med accent-farge på numre

### Content

**Section Header:**
```
H2: Slik fungerer det
Subtext: Fra null til bidrag på under 2 minutter.
```

**Step 1:**
```
Nummer: 01
Ikon: UserPlus
Tittel: Opprett konto
Beskrivelse: Registrer deg gratis med GitHub, Google, eller e-post.
```

**Step 2:**
```
Nummer: 02
Ikon: Upload
Tittel: Del eller utforsk
Beskrivelse: Post ditt første snippet eller bla gjennom andres bidrag.
```

**Step 3:**
```
Nummer: 03
Ikon: Zap
Tittel: Løs challenges
Beskrivelse: Velg en challenge, last ned data, og submit svaret ditt.
```

## 4. Challenge Preview

### Layout
- Showcase av en faktisk challenge (eller mock)
- Venstre: Challenge-beskrivelse
- Høyre: Kode-editor med eksempel-data
- CTA-knapp under

### Content

**Section Header:**
```
H2: Dagens Challenge
Badge: MEDIUM · 25 POENG
```

**Challenge Title:**
```
H3: Find the Missing Number
```

**Challenge Description (forkortet):**
```
En tallfølge mangler et tall. Gitt en liste fra 1 til N 
der ett tall mangler, finn det manglende tallet.

Eksempel: [1, 2, 4, 5] → Svar: 3
```

**CTA Button:**
```
Prøv denne challengen →
```

### Bilde-behov

**Code Preview:**
- Mockup av challenge-detalj side
- Viser eksempel-data i kode-blokk
- Viser difficulty badge
- Format: Screenshot eller component mockup

- Full-width mørk seksjon
- 3-4 store tall med labels
- Subtil animasjon på tall (count-up effekt)

### Content

**Section Style:** Ren statistikk, ingen header

**Stat 1:**
```
Tall: 500+
Label: Kode-snippets delt
```

**Stat 2:**
```
Tall: 150+
Label: Challenges løst
```

**Stat 3:**
```
Tall: 1000+
Label: Diskusjonssvar
```

**Stat 4 (valgfritt):**
```
Tall: 50+
Label: Aktive utviklere
```

> **Note for utvikler:** Disse tallene kan starte lavt og oppdateres 
> etterhvert, eller vises som "Kom i gang tidlig"-variant med 
> "Bli en av de første" messaging.

```
H2: Bli en early adopter
Subtext: DevHub er i beta - vær blant de første som former 
plattformen. Dine bidrag teller ekstra!

CTA: Registrer deg nå →
```

## Implementasjons-notater for Utvikler

### Payload CMS Blocks

Opprett følgende block-typer i Payload:

1. **HeroBlock**
   - badge (text, optional)
   - headline (text, required)
   - subheadline (textarea)
   - primaryCTA (group: text, url)
   - secondaryCTA (group: text, url, optional)
   - heroImage (upload, optional)

2. **FeaturesGridBlock**
   - sectionTitle (text)
   - sectionSubtext (text)
   - features (array of: icon, title, description)

3. **HowItWorksBlock**
   - sectionTitle (text)
   - sectionSubtext (text)
   - steps (array of: number, icon, title, description)

4. **ChallengePreviewBlock**
   - challengeId (relationship to Challenge, optional)
   - fallbackTitle (text)
   - fallbackDescription (textarea)

5. **StatsBlock**
   - stats (array of: value, label)
   - alternativeMode (boolean - for "early adopter" variant)

6. **CTABlock**
   - headline (text)
   - subtext (textarea)
   - primaryCTA (group: text, url)
   - secondaryText (text, optional)

### Responsiv-tilpasning

| Blokk | Desktop | Tablet | Mobil |
|-------|---------|--------|-------|
| Hero | Side-by-side | Stacked | Stacked |
| Features | 3 cols | 2 cols | 1 col |
| How It Works | Horizontal | Horizontal | Vertical |
| Stats | 4 cols | 2x2 | 2x2 |
